## Структура датасета FUNSD

### Общая структура
ДатасетFUNSD организован по стандартной схеме:
```
FUNSD/
├── training_data/        # Обучающая выборка
│   ├── annotations/      # JSON-файлы с аннотациями (формы)
│   └── images/           # PNG-изображения документов
└── testing_data/         # Тестовая выборка
    ├── annotations/
    └── images/
```

### Структура файла аннотации (.json)

Каждый annotation-файл содержит объект с ключом `"form"` - массив объектов, представляющих элементы формы документа:

**Основные поля каждого элемента:**
- `box`: [x1, y1, x2, y2] - координаты прямоугольной области (левый верхний и правый нижний углы)
- `text` - текстовое содержимое элемента
- `label` - тип элемента: `"question"`, `"answer"`, `"other"`, `"header"`
- `words`: массив подэлементов с собственными `box` и `text`
- `id` - уникальный идентификатор элемента
- `linking` - связи между элементами (пары ID)

### Типы меток (labels)
- **question** - вопросы/подписи полей (например, "TO:", "DATE:", "SIGNATURE(S)")
- **answer** - ответы/данные в полях (например, телефонные номера, даты, значения)
- **other** - служебный контент или пустые области
- **header** - заголовки разделов

### Особенности данных
- Файлы аннотаций и изображений имеют соответствие по именам (например: `00040534.json` ↔ `00040534.png`)
- Датасет содержит 167 обучающих и 49 тестовых документов
- Формат координат в `box`: [x1, y1, x2, y2] (левый верхний и правый нижний углы)
